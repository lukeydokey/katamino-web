# Katamino 플레이 경험 개선안

## 목적

이 문서는 "레거시 게임을 웹으로 옮긴다"가 아니라, **일반 사용자가 실제로 들어와서 쉽게 이해하고 재미있게 반복 플레이할 수 있게 만드는 것**을 목표로 정리한 개선안이다.

## 현재 상태 요약

- 방 생성 / 참가 / 시작의 기본 흐름은 존재한다.
- 로컬 보드에서는 블록 선택, 회전, 배치, 리셋이 가능하다.
- 공유 룸에서 기본적인 보드 상태 동기화와 수 반영은 가능하다.
- 기권과 리매치 흐름도 들어가 있지만, 전체 경험은 아직 제품 완성 단계라기보다 확장 중인 상태다.

## 핵심 문제

### 1. 첫 진입 이해도가 낮다
- Katamino를 모르는 사용자는 목표를 바로 이해하기 어렵다.
- 방에 들어가기 전 무엇을 해야 하는지, 어떻게 이기는지, 왜 재미있는지가 약하다.

### 2. 대기 경험이 심심하다
- 룸에 들어간 뒤 상대를 기다리는 동안 할 일이 거의 없다.
- 준비 상태, 상대 접속 상태, 시작 조건이 눈에 잘 안 들어오면 사용자는 멈춘다.

### 3. 승부 흐름이 아직 약하다
- 현재는 보드를 놓는 행위는 가능하지만, 긴장감 있는 턴 흐름과 승패 감각은 아직 부족하다.
- 시작 조건, 턴 진행, 실수 피드백, 종료 경험이 사용자 기준으로 더 정리되어야 한다.

### 4. 다시 플레이하고 싶게 만드는 장치가 약하다
- 한 판이 끝난 뒤 즉시 재대결, 리매치, 기록 확인, 가벼운 성취감 같은 요소가 없다.

## 개선 방향

## 1. 홈 화면 개선

### 목표
- 사용자가 첫 화면에서 바로 "무슨 게임인지"와 "어떻게 시작하는지"를 이해하게 만든다.

### 개선안
- 상단 핵심 설명을 한 줄로 단순화
  - 예: "블록을 먼저 배치할 위치를 읽고, 상대보다 먼저 공간을 완성하세요."
- `방 만들기`와 `방 참가하기`를 가장 강한 CTA로 유지
- 로컬 연습은 보조 행동으로 배치
- 짧은 규칙 안내 3줄 추가
  - 블록은 회전 가능
  - 놓을 수 없는 위치는 미리 표시
  - 상대와 번갈아 배치

## 2. 룸/로비 개선

### 목표
- "기다리는 화면"이 아니라 "곧 게임이 시작될 공간"처럼 느껴지게 만든다.

### 개선안
- 룸 코드 복사 버튼 추가
- 참가자 2석을 시각적으로 표시
  - HOST / GUEST 또는 플레이어 1 / 플레이어 2
- 접속 상태 표시
  - 입장 대기 중
  - 상대 접속 완료
  - 시작 가능
- `게임 시작` 버튼은 host만 보이게 하거나, guest에겐 비활성 사유를 보여주기
- 리매치/다시 하기 흐름을 같은 위치에서 이어지게 설계

## 3. 실제 게임 화면 개선

### 목표
- 조작이 자연스럽고, 다음 행동이 항상 보이도록 만든다.

### 개선안
- 현재 적용한 미리보기 배치 UX를 유지하되, 다음을 추가 검토
  - 선택 중인 블록 강조
  - 놓을 수 없는 위치일 때 더 명확한 시각 피드백
  - 모바일에서도 비슷한 수준의 조작 이해도 보장
- 선택 해제 방식 유지
  - 같은 블록 다시 누르기
  - `Esc`
- 향후 드래그 UX를 검토하더라도, 현재 hover 미리보기 방식은 유지하는 편이 안전함

## 4. 턴/승패 경험 개선

### 목표
- 사용자가 "내 차례인지", "지금 유리한지", "왜 끝났는지"를 바로 이해하게 만든다.

### 개선안
- 현재 턴 플레이어 표시를 더 강하게 표현
- 턴 타이머 재도입 여부 검토
  - 너무 빠른 제한은 진입 장벽
  - 너무 느리면 긴장감 부족
- 종료 시 결과 화면 필요
  - 승리
  - 패배
  - 기권
  - 리매치 제안

## 5. 리텐션(다시 플레이) 장치

### 목표
- 한 판 끝난 후 자연스럽게 다음 행동으로 이어지게 만든다.

### 개선안
- `같은 상대와 다시 하기`
- `새 룸 만들기`
- 간단한 전적 표시
  - 최소한 같은 세션 안에서는 승/패 수를 보여줄 수 있음

## 6. 불필요한 설명 제거 원칙

실제 사용자 화면에서는 다음 종류의 문구를 제거하거나 숨긴다.

- "스캐폴드"
- "검증용 UI"
- "다음 단계에서 추가됩니다"
- "레거시 WinForms 기준"
- 내부 구조 설명 (`Supabase`, `Vercel`, `room lifecycle` 등)

이 정보는 사용자 화면이 아니라 README 또는 내부 문서에만 남긴다.

## 우선순위

### P1
- 공유 보드에서 블록을 숫자 대신 색상으로 구분
- 방 생성 시 턴 제한 시간 설정 추가
- 관전자 모드 추가

### P2
- 공유 룸 채팅 추가
- 홈 화면 규칙 안내 정리
- 모바일 대응 개선

### P3
- 전적, 기록, 반복 플레이 유도 장치
- 사운드/시각 효과 강화
