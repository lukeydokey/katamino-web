# AGENTS.md

## 프로젝트 요약
- 목표: 예전 1:1 Katamino 데스크톱 게임을 웹 앱으로 재구현한다.
- 레거시 참고 원본은 `docs/temp/Katamino_v12/` 아래에 있으며, 읽기 전용 참고 자료로 취급한다.
- 구현 완료 후 배포 대상은 Vercel + Supabase이다.

## 작업 원칙
- 게임 규칙은 먼저 프레임워크와 분리된 TypeScript 모듈로 정리한다.
- 레거시 구현을 줄 단위로 그대로 옮기기보다, 의도된 게임 규칙을 보존하는 방향으로 맞춘다. 명확히 검증된 레거시 버그는 수정할 수 있다.
- v1 아키텍처는 단순하게 유지한다. 웹 클라이언트 + 최소한의 백엔드/Realtime 구성을 우선하며, 과한 인프라는 피한다.
- 문서, 계획서, 주석은 가능하면 한글을 우선 사용한다. 다만 라이브러리/API 이름, 코드 심볼, CLI 명령어는 원문 표기를 유지한다.

## 계획 및 문서
- 작업 계획 문서는 보통 `docs/plan/` 아래에 작성한다.
- `docs/temp/`와 `docs/plan/`은 이 저장소에서 의도적으로 gitignore 대상에 포함한다.
- 레거시 앱을 참고할 때는 `Game_form.cs`, `Block.cs`, `DiceGame.cs`를 먼저 읽는다.

## 현재 기대하는 목표 구조
- 프론트엔드/앱 셸은 Vercel에 배포하는 Next.js를 우선 기준으로 본다.
- 백엔드 서비스는 Supabase Postgres + Realtime 조합을 기준으로 하며, 초기에는 guest 수준의 가벼운 식별 방식을 우선 고려한다.
- 멀티플레이는 웹 전송 방식에 맞게 다시 설계해야 하며, 레거시 TCP loopback 모델을 그대로 포팅하지 않는다.
